## General Guidelines

- When writing or editing long Markdown files, put each full sentence on its own line. Preserve normal Markdown structure, but avoid wrapping multiple sentences onto one physical line.
- When writing or editing LaTeX code, prefer to use available shortcuts and macros.
- When making technical decisions, do not give much weight to development cost. Instead, prioritize quality, simplicity, robustness, scalability, and long-term maintainability.
- When making bug fixes, always start with reproducing the bug with an isolated unit test, and optionally an E2E test where possible. This makes sure you find the real problem so your fix will actually solve it.
